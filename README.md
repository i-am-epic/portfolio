# Bento Portfolio

A modern, customizable portfolio website built with React, Next.js, and SCSS. Features a beautiful bento grid layout that's fully responsive.
# Website
https://nikboson.vercel.app

## Features

- Clean and modern design with bento grid layout
- Fully responsive for all screen sizes
- Dark mode support
- Simple and modular component structure
- Easy to customize with SCSS variables
- Built with React and Next.js
- No external UI libraries - pure React components
- Gemini-powered portfolio chatbot with lightweight RAG retrieval
- Live preview links for featured projects

## 🎮 3D Minecraft-style World (`/world`)

The site now opens into an explorable, first-person, Minecraft-themed 3D world built
with **three.js + @react-three/fiber + drei**. The original bento portfolio is fully
preserved at **`/classic`** and is used as an automatic fallback on mobile / low-power /
no-WebGL devices (and as a "Skip to classic site" button).

- `/` → redirects to `/world`
- `/world` → the 3D world (desktop). SSR-renders a hidden, crawlable summary of all
  projects + links for SEO, and falls back to `/classic` when 3D isn't supported.
- `/classic` → the original bento portfolio (unchanged)
- `/works`, `/contact` → unchanged

### Controls
- **WASD** move · **mouse** look (click to lock the pointer) · **Shift** sprint ·
  **Space** jump · **E** interact with the highlighted object · **1–9** quick menu ·
  **Esc** release pointer / close panel

### What's in the world
- **Spawn plaza** with a welcome sign, the **NAVI villager** (talk to it — wired to the
  existing Gemini RAG at `/api/profile-chat`), a **Jukebox** (live Spotify via `/api/spotify`),
  a **Character Build** board, an **Impact Board**, and a **Patch Notes** lectern.
- **Project Hall** — 17 project **chests**; walk up and press **E** to open a chest and read
  the project (tags, description, GitHub + live-preview links).
- **Contact Portal** at the far end.
- **🎰 Lucky Blocks slot machine** — a fully playable, Minecraft-themed slot game (see below).

### 🎰 Lucky Blocks slot machine
A modern slot game embedded in the world (hotbar **8**, or walk up to the cabinet and press **E**):
- 3 reels with staggered stops, weighted symbols, and pair / triple / **JACKPOT** payouts.
- Synthesized **WebAudio** sound effects (lever, reel ticks, win chime, jackpot fanfare) — no
  audio files; toggle with the 🔊 button.
- Win **effects**: confetti, screen shake, and a gold jackpot glow.
- On a big win (≥150 or a 💎 jackpot) you **carve a name card**; cards, the all-time
  **high score**, and your coin purse persist in `localStorage`.
- **🏆 Hall of Fame** leaderboard inside the machine.

> Storage note: the leaderboard is currently per-browser (`localStorage`). It can be
> upgraded to a shared backend (e.g. a Next.js API route + KV/DB) without changing callers —
> see `lib/world/leaderboard.ts`.

## 🍾 Scroll Feedback Wall (all pages)

A "message in a bottle" guestbook that's visible on **every** page (classic + world).
Anyone can throw a scroll (with a name or anonymously); the bottles stack along the
bottom of the classic site and open in a panel inside the 3D world. Click a bottle to
read the scroll on a parchment.

- Floating **🍾 Throw a scroll** + **📜 Scrolls** buttons are always present.
- Submissions are sanitized + lightly profanity-masked (`lib/server/moderate.ts`).
- Bottle/parchment art is original CC0 pixel art (`scripts/gen-scroll-art.cjs` → `public/scroll/`).

### Shared storage (Upstash Redis)
The wall and the slot **leaderboard** are shared across all visitors via Upstash Redis:

- API routes: `app/api/scrolls/route.ts`, `app/api/leaderboard/route.ts`
- Storage layer: `lib/server/store.ts` — uses Upstash REST when
  `UPSTASH_REDIS_REST_URL` + `UPSTASH_REDIS_REST_TOKEN` are set, otherwise falls back
  to a local `.data/` JSON file (dev only, not shared).

**To enable the shared wall in production:** create a free database at
[console.upstash.com](https://console.upstash.com/), then set those two env vars
(locally in `.env.local`, and in your Vercel project settings). Without them everything
still works locally via the file fallback (and the client also keeps a `localStorage`
mirror so a thrower always sees their own bottle).

The slot machine is also available on the **classic** site via a floating 🎰 launcher
(it lives in the world's hotbar there). Coins/high-score stay per-browser; jackpot name
cards post to the shared leaderboard.

## Customization

### Colors and Theme

Edit `src/styles/base.scss` to customize:
- Color scheme
- Typography
- Dark mode colors
- Global styles

### Components

Each component has its own SCSS file in `src/styles/components/`. Customize individual components by editing their respective style files.

### Content

Update the content by modifying the components in `src/components/`. Each component is self-contained and easy to customize.

## Getting Started

1. Clone the repository:
```bash
git clone https://github.com/yourusername/bento-portfolio.git
```

2. Install dependencies:
```bash
npm install
```

3. Configure environment variables:
```bash
cp .env.example .env.local
```

Set at least:
- `GEMINI_API_KEY` for `/api/profile-chat`
- Spotify keys for `/api/spotify` (optional)

4. Run the development server:
```bash
npm run dev
```

5. Open [http://localhost:3000](http://localhost:3000) to view your portfolio.
or visit "https://nikboson.vercel.app"

## Building for Production

```bash
npm run build
npm start
```
